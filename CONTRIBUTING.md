## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:
- **Description** of the bug
- **Steps to reproduce**
- **Expected behavior**
- **Actual behavior**
- **System information** (Pi model, OS version)
- **Log output** (from bible_display.log)

### Suggesting Features

Feature requests are welcome! Please:
- Check if it's already been suggested
- Describe the feature clearly
- Explain why it would be useful
- Consider if it fits the project scope

### Submitting Pull Requests

1. **Fork the repository**
2. **Create a branch** (`git checkout -b feature/amazing-feature`)
3. **Make your changes**
4. **Test thoroughly** on actual hardware if possible
5. **Commit** (`git commit -m 'Add amazing feature'`)
6. **Push** (`git push origin feature/amazing-feature`)
7. **Open a Pull Request**

### Code Guidelines

- **Python style**: Follow PEP 8
- **Comments**: Add comments for complex logic
- **Documentation**: Update relevant .md files
- **Testing**: Test on Raspberry Pi if possible

### Documentation Contributions

Documentation improvements are highly valued!
- Fix typos and errors
- Improve clarity
- Add examples
- Translate to other languages

### What We're Looking For

- **Bug fixes** - Always welcome
- **Performance improvements**
- **Better error handling**
- **Additional Bible translations**
- **Hardware compatibility** (other e-paper displays)
- **Documentation improvements**
- **Example configurations**
- **Photos of your setup** (for gallery)

### What We're NOT Looking For

- Features requiring internet connection
- Features that increase complexity significantly
- Breaking changes without good reason
- Proprietary/copyrighted Bible translations

## Development Setup

1. Clone the repository
2. Set up hardware (Raspberry Pi + e-paper display)
3. Run `./setup.sh`
4. Make your changes
5. Test with `python3 bible_display.py`
